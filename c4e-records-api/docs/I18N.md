# i18n en c4e-records-api

Este proyecto usa `c4e-i18n-core` para gestionar internacionalización.

## Estándares

→ **Lee primero:** [I18N_STANDARDS.md](../../c4e-i18n-core/docs/I18N_STANDARDS.md)

## Estructura en este proyecto

```
c4e-records-api/
├── common/
│   └── RecordsApiCommonMessageKey.java     (todas las claves del dominio)
├── src/main/resources/
│   └── messages.properties
```

## Convención de Claves

- `incident.*` — Validaciones y errores del dominio incident
- `medidaqh.*` — Validaciones y errores de medidas
- `user.*` — Errores relacionados a usuarios
- `validation.*` — Validaciones genéricas
- `error.*` — Errores de sistema
- `endpoint.*` — Mensajes de endpoints

## Ejemplo: Agregar un Nuevo Mensaje

### Paso 1: Agregar Clave al Enum

```java
// En RecordsApiCommonMessageKey.java
INCIDENT_PAYLOAD_JSON_PARSE_ERROR("incident.payload.json.parse.error")
```

### Paso 2: Agregar Entrada en Properties

```properties
# En messages.properties
incident.payload.json.parse.error=Payload is not valid JSON, storing as plain text: {0}
```

### Paso 3: Usar en el Código

```java
import com.com4energy.i18n.core.Messages;
import com.com4energy.recordsapi.common.RecordsApiCommonMessageKey;

// Sin parámetros
String msg = Messages.get(RecordsApiCommonMessageKey.SOME_KEY);

// Con parámetros
log.warn(Messages.format(RecordsApiCommonMessageKey.INCIDENT_PAYLOAD_JSON_PARSE_ERROR, e.getMessage()));
```

## Principales

- ✅ **Siempre usar enums** en lugar de strings hardcodeados
- ✅ **Agrupar por dominio** en el enum (comentarios)
- ✅ **Usar convención de prefijos** para evitar colisiones
- ❌ **Nunca** poner strings mágicos en logs/excepciones

Más detalles en [I18N_STANDARDS.md](../../c4e-i18n-core/docs/I18N_STANDARDS.md).

# i18n en c4e-event-publisher

Este proyecto usa `c4e-i18n-core` para gestionar internacionalización.

## Estándares

→ **Lee primero:** [I18N_STANDARDS.md](../../c4e-i18n-core/docs/I18N_STANDARDS.md)

## Estructura en este proyecto

```
c4e-event-publisher/
├── common/
│   └── PublisherCommonMessageKey.java      (claves compartidas entre publishers)
├── incident/publisher/
│   └── IncidentPublisherMessageKey.java    (claves específicas de incident)
└── src/main/resources/
    └── messages.properties
```

## Convención de Claves

- `incident.publisher.*` — Claves de IncidentPublisher
- `publisher.*` — Claves compartidas (futuro)

## Ejemplo

```java
// En IncidentPublisher.java
import com.com4energy.i18n.core.Messages;
import com.com4energy.event.publisher.incident.publisher.IncidentPublisherMessageKey;

Messages.format(IncidentPublisherMessageKey.INCIDENT_TYPE_NOT_CONFIGURED, type.key())
```

## Agregar una Nueva Clave

1. Agregar enum a `IncidentPublisherMessageKey` (o crear nuevo si es para otro publisher)
2. Agregar entrada en `messages.properties`
3. Usar en el código con `Messages.get()` o `Messages.format()`

Más detalles en [I18N_STANDARDS.md](../../c4e-i18n-core/docs/I18N_STANDARDS.md).
